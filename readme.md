## Использование

### Подготовка данных
Создайте файл `data/data.txt` в следующем формате:
```
Шкаф	Сигнал	Откуда	Куда
1HV19	0501	XT11-b9	XT10-b9
1HV19	0501	XT11-b9	XT12-b9
2HV20	0601	XTK1-b1	XTK2-b1
```

### Запуск приложения
```bash
# Основной способ запуска
python main.py

# С указанием пользовательских путей (если требуется модификация)
python main.py --source ./data/custom_data.txt --target ./output/custom_result.txt
```

### Формат входных данных
Файл данных должен быть в формате TSV (табуляция как разделитель) с колонками:
- Шкаф (cabinet)
- Сигнал (signal)
- Откуда (from terminal)
- Куда (to terminal)

## Функциональность

### Основные модули

1. **DataParser** - парсит входные данные из TSV-файла
   - Определяет шкафы и соединения
   - Сохраняет информацию о происхождении данных (файл-строка)

2. **DataMerging** - объединяет связанные соединения
   - Использует алгоритм Union-Find для группировки связанных терминалов
   - Создает группы перемычек внутри каждого шкафа

3. **DataWriter** - выводит результаты
   - Сортирует шкафы по номеру
   - Сортирует терминалы по специальному алгоритму
   - Выводит информацию о происхождении данных для каждого терминала

### Алгоритм сортировки терминалов
Функция `sorting_key` обеспечивает специальный порядок сортировки:
1. **XTK** - высший приоритет
2. **XT** (кроме XTN) - средний приоритет  
3. **XTN** - низший приоритет
4. Остальные терминалы

Внутри групп сортировка по числовым значениям, извлеченным из названий терминалов.

## Пример вывода

```
1HV19
     XT10-b9    1_1
     XT11-b9    1_1, 2_1
     XT12-b9    2_1
```

Где:
- `1HV19` - номер шкафа
- `XT10-b9`, `XT11-b9`, `XT12-b9` - терминалы
- `1_1`, `2_1` - ссылки на исходные данные (файл_строка)

## Разработка

### Архитектура
Проект использует паттерн "Обработчик данных" с четким разделением ответственности:
- **Парсинг** → **Объединение** → **Запись**

2. Обновите класс `Application` при необходимости
3. Протестируйте на различных наборах данных

## Ключевые особенности проекта:

1. **Обработка электрических соединений** - специализирован для работы с клеммами шкафов управления
2. **Алгоритм Union-Find** - для эффективного объединения связанных соединений
3. **Кастомная сортировка** - специальный порядок обработки терминалов разных типов
4. **Трассируемость** - сохранение информации о происхождении данных
5. **Модульная архитектура** - четкое разделение ответственности между компонентами
